设计方案：引擎化改造

将"钢铁意志"从硬编码游戏改造为 JSON 驱动的通用互动叙事引擎

📋 需求确认

✅ JSON 数据文件编写 DLC（创作者无需懂 C++）
✅ 同一玩法框架，任意题材
✅ DLC = 文件夹形式，放入 dlc/ 目录自动加载
✅ 现有 5 个战役转为 JSON DLC，引擎纯净化
✅ 最小可行：剧本数据 + 职业定义 + 背景音乐
✅ 每个 DLC 独立定义职业，互不干扰
✅ 一个 DLC = 一个完整战役（多个章节）
✅ 章节间支持分支路线

🏗️ 架构对比

❌ 改造前
main.cpp
├── GameEngine
│ ├── registerScenarios() ← 硬编码 new 5个类
│ ├── ScenarioId 枚举 ← 硬编码
│ └── PlayerClass 枚举 ← 硬编码
├── scenarios/ ← 每个场景一个 .h/.cpp
└── ui/ ← Qt Widgets 固定界面

新增场景 = 写C++ + 改CMake + 重编译
✅ 改造后
main.cpp
├── GameEngine (纯核心引擎)
│ ├── DlcManager ← 扫描 dlc/ 目录
│ ├── NodeEngine ← 节点导航+选择+骰子
│ ├── PlayerSystem ← 动态职业属性
│ └── SaveSystem ← JSON存档
├── dlc/ ← 创作者只碰这里
│ ├── third-reich/ ← 官方DLC示例
│ │ ├── manifest.json
│ │ ├── chapters/*.json
│ │ └── music/*.mp3
│ └── three-kingdoms/ ← 第三方DLC
├── engine/ ← 通用引擎模块
└── ui/ ← 通用UI框架

新增DLC = 写JSON + 丢进文件夹 → 搞定

🧱 DLC 文件夹结构设计

dlc/three-kingdoms/ 示例
