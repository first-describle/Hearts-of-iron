🖥️ UI 适配要点

界面 当前行为 改造后行为
主菜单 "新游戏"→ 直接创建角色 新增"选择DLC"步骤：先选DLC → 再创建角色（职业列表来自该DLC）
角色创建 6个固定职业按钮 动态生成：根据DLC manifest.classes[] 生成对应数量的职业卡片
场景选择 ScenarioId枚举的5个场景 不再需要：章节解锁由 DLC JSON 中 unlock 字段控制，线性推进（后续可扩展分支）
游戏界面 从scenarios/枚举获取节点 从 NodeEngine 获取节点，NodeEngine 从 DLC JSON 加载
存档界面 保存 Player 对象 保存 PlayerSystem 状态 + dlcId 字段（知道该存档属于哪个DLC）

🎯 创作者工作流

1. 在 dlc/ 下新建文件夹，命名如 dlc/three_kingdoms/
2. 编写 manifest.json（复制官方示例改）
3. 在 chapters/ 下编写各章节 JSON（复制官方示例改）
4. 放入 music/ 音频文件
5. 启动游戏 → 引擎自动扫描加载 → 即可在菜单中看到新 DLC
