🧩 DlcManager
职责：扫描 + 加载 + 校验 DLC
• 启动时扫描 dlc/ 目录
• 解析每个 DLC 的 manifest.json
• 校验 JSON 合法性
• 提供已加载的 DLC 列表给 UI
• 与原模块对应：
← 替代 registerScenarios()
← 替代 ScenarioId 枚举
🧩 NodeEngine
职责：故事节点导航 + 选择执行
• 当前节点定位
• 执行 Choice（骰子/叙事/分支）
• 章节间跳转
• 标志位检查
• 与原模块对应：
← GameEngine 的场景导航部分
← StoryNode/Choice 数据结构保留
🧩 PlayerSystem
职责：动态职业 + 属性管理
• 从 DLC manifest 读职业定义
• 属性 HP/士气/etc
• 标志位集合
• 存档序列化
• 与原模块对应：
← Player 类重构为动态职业
← PlayerClass 枚举移除
🧩 DiceSystem
职责：骰子判定（无变化）
• D100 随机数
• 职业加成判定
• 阈值比较
• 与原模块对应：
← GameEngine::rollDice()

关键变化

旧结构 新结构
PlayerClass 枚举 - 6个固定职业 DlcClass 结构体 - DLC的JSON定义，包含 id/name/bonusType
ScenarioId 枚举 - 5个固定场景 DlcManifest - DLC元数据对象，包含多个Chapter
registerScenarios() 硬编码注册 DlcManager::scanDirectory() 自动扫描
场景 C++ 类 (ScenarioBase 子类) Chapter JSON 文件 (DlcChapter 运行时对象)
classAllowedInScenario() 硬编码映射 DLC JSON 中 classRestricted 字段 + allowedClasses
